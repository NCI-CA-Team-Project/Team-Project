# Team-Project
10-Weeks Progress
------------------------------------------------------------------------------------------------
Week 1 – Start of Project
The team was formed and everyone got their roles.

We decided what the app would do a language learning app with chat, stats, friend requests, and support for 4 languages.

We chose our tech stack: Frontend, Backend, AWS, RDS database, and WebSockets for live chat.

We set up the code repository and completed the initial project planning.
------------------------------------------------------------------------------------------------
Week 2 – Requirements & Design
We wrote down everything the app must do (functional requirements) and quality goals (non-functional requirements).

We drew wireframes and UI mockups so everyone could see how the app would look.

We designed the database schema for users, messages, stats, and requests.

We planned all the API endpoints (how frontend and backend would talk to each other).
------------------------------------------------------------------------------------------------
Week 3 – Environment Setup
We configured the AWS environment.

We started the backend boilerplate code (the basic structure).

We made the frontend scaffolding (basic page structure).

We set up development environments for every team member so everyone could work.
------------------------------------------------------------------------------------------------
Week 4 – Core Backend Development
We built user login and signup (authentication logic).

We made basic CRUD operations (Create, Read, Update, Delete) for user data.

We connected the database to the backend and tested it locally.

We did API testing using Postman.
------------------------------------------------------------------------------------------------
Week 5 – Core Frontend Development
We made the login and registration screens.

We made a basic chat interface.

We connected the frontend to the backend.

We got the first successful user data flow working locally.
------------------------------------------------------------------------------------------------
Week 6 – Integration & AWS Deployment (First Sprint)
We deployed the AWS RDS database to the cloud.

We deployed the backend to AWS.

We deployed the frontend to the cloud.

We found some integration issues Cameron noted that frontend, backend, and RDS needed to be linked much earlier.
------------------------------------------------------------------------------------------------
Week 7 – Mid-Point Preparation
Martin drafted the mid-point presentation slides.

The team set a goal to have a working prototype by Wednesday.

Cameron pushed the integration work further and made good progress.

AWS user data was successfully sent from the frontend all the way into the database.

Big milestone achieved we had a working prototype just in time for the presentation.
------------------------------------------------------------------------------------------------
Week 8 – Mid-Point Presentation & Sprint 2 Retrospective
March 9 – We delivered the mid-point presentation.

Martin presented remotely via Teams.

We showed a live demo of the working prototype.

testing confirmed that everything was fully integrated.

We held a Sprint 2 retrospective the team talked about what went well and what could be better.

We set clear goals for Sprint 3: get full integration done early, and make sure connect and chat features are working properly.
------------------------------------------------------------------------------------------------
Week 9 – Sprint 3 Development
Cameron pushed updated versions of the frontend, backend, and central database.

We assigned the final remaining tasks to everyone:

WebSocket integration (for live real-time chat)

Stats feature (track user progress)

Requests feature (send and accept friend requests)

Language dropdown menu (support 4 different languages)

Vincentas offered to help the team and took responsibility for white box and black box testing.

We also started tidying up the code to make it clean and readable.
------------------------------------------------------------------------------------------------
Week 10 – Final Week & Submission
March 27 – Martin announced that this was the last week of the project.

We held a final team meeting to prioritise realistic targets we knew we only had one week left.

All remaining features were completed and tested thoroughly.

Vincentas finished the white box and black box testing.

The final code was pushed to lingoBeta.

The project was submitted on time
